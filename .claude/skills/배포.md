# 배포 스킬

사용자가 "배포해줘"라고 하면 아래 순서를 따른다.

## 1단계 — 현재 버전 확인 후 목표 버전 질문

`apps/desktop/package.json`의 `version` 필드를 읽어 현재 버전을 확인한다.

AskUserQuestion 도구로 질문한다:
- 질문: "현재 버전은 {현재버전}입니다. 어떤 버전으로 올릴까요?"
- 옵션: patch / minor / major / 직접 입력

## 2단계 — 목표 버전 계산

- **patch**: `x.y.z` → `x.y.(z+1)`
- **minor**: `x.y.z` → `x.(y+1).0`
- **major**: `x.y.z` → `(x+1).0.0`
- 직접 입력한 경우 그 값을 사용

## 3단계 — git 히스토리 조회

```bash
# 마지막 태그 확인
git describe --tags --abbrev=0 2>/dev/null || echo "NONE"
```

- 태그가 있으면: `git log {last_tag}..HEAD --pretty=format:"%s" --no-merges`
- 없으면: `git log --pretty=format:"%s" --no-merges`

## 4단계 — 패치노트 생성 및 확인

커밋 메시지를 분석해 분류한다:
- `feat` → **새 기능**
- `fix` → **버그 수정**
- `refactor` / `perf` / `chore` / `build` → **개선**
- `docs` → 생략

prefix(`feat(crawler):` 등)를 제거하고 한국어로 자연스럽게 정리한다. 빈 섹션은 생략한다.

```markdown
# v{목표버전} 패치노트

## 새 기능
- ...

## 개선
- ...

## 버그 수정
- ...
```

생성한 패치노트를 사용자에게 보여준다. 수정 요청이 있으면 반영한 뒤 다음 단계로 넘어간다.

## 5단계 — 파일 수정

1. `apps/desktop/package.json`의 `version`을 목표 버전으로 수정
2. `apps/desktop/src/renderer/src/features/patch-notes/assets/{목표버전}.md` 생성

## 6단계 — commit, tag, push

```bash
git add apps/desktop/package.json
git add apps/desktop/src/renderer/src/features/patch-notes/assets/{목표버전}.md
git commit -m "chore: release v{목표버전}"
git tag v{목표버전}
git push origin main
git push origin v{목표버전}
```

## 7단계 — 빌드

```bash
pnpm --filter @repo/desktop build
```

빌드 성공 시 완료를 안내한다. 실패 시 오류를 보여주고 멈춘다.
